# ADR-001: 리팩토링 전략

## 상태
- [x] 승인됨 (2024-02)

## 맥락
레거시 코드를 안전하게 리팩토링해야 한다. 리팩토링 과정에서 기존 기능이 깨지지 않도록 보장하면서, 코드 품질을 개선해야 한다.

## 선택지

### 1. Big Bang 리팩토링
전체 코드를 한 번에 재작성한다.
- 장점: 깔끔한 설계 가능
- 단점: 위험도 높음, 테스트 어려움, 롤백 불가

### 2. 점진적 리팩토링 (선택)
작은 단위로 나누어 단계별로 진행한다.
- 장점: 위험도 낮음, 각 단계 검증 가능, 롤백 용이
- 단점: 시간 소요, 중간 상태 관리 필요

## 결정
**점진적 리팩토링**을 선택하고, 다음 4단계로 진행한다:

1. **코드 분석**: 문제점 식별 및 문서화
2. **인수 테스트 작성**: 현재 작동을 보존하는 안전망 확보
3. **서비스 계층 추출**: 구조 변경 (작동 변경 없음)
4. **코드 스멜 수정**: 품질 개선 (작동 변경 없음)

## 핵심 원칙
- **구조 변경과 작동 변경을 섞지 않는다**
- **한 번에 한 조각만 바꾼다**
- **커밋은 논리 단위로, 설명 가능한 diff로만 한다**

## 근거
- 75개 인수 테스트가 안전망 역할을 하여, 변경 후 기능 보존을 검증할 수 있다
- 작은 커밋 단위로 문제 발생 시 빠른 원인 파악과 롤백이 가능하다
- 구조 변경과 작동 변경을 분리하면 diff 리뷰가 명확해진다

## 결과
- 4단계 리팩토링 완료 (구조 변경 단계)
- 모든 75개 테스트 통과
- 다음 단계(5단계: 작동 변경)로 진행 가능
